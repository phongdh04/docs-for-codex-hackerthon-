# Workflow: Frontend Development Cycle

## Stage 1: Design Review (Rà soát thiết kế)
- Đọc `ui_ux_spec.md` và các ảnh thiết kế (nếu có).
- Đối chiếu với `user-flow.md` để hiểu logic chuyển trang.
- Tạo folder task tại `task-fe-level/task-[ID]`.
- Viết `task-todo.md` và `acceptance_criteria.md`.

## Stage 2: Skeleton & Styling (Dựng khung & Style)
- Tạo cấu trúc Component.
- Viết CSS/SCSS để đạt độ chính xác Pixel Perfect.
- Đảm bảo tính Responsive.

## Stage 3: Logic & Integration (Logic & Tích hợp)
- Viết TypeScript xử lý sự kiện người dùng.
- Mock dữ liệu từ API Contract để chạy thử UI.
- Tích hợp với Backend (Kevin) khi đã sẵn sàng.

## Stage 4: Testing & Handover (Kiểm thử & Bàn giao)
- Tự kiểm tra UI trên các trình duyệt phổ biến.
- Submit cho Bob review code.
- Phối hợp với Sarah (QC) để fix các lỗi về hiển thị.
