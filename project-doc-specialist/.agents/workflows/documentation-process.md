# Workflow: Standard Project Documentation Process

## Purpose
Đảm bảo quy trình từ lúc nhận yêu cầu đến khi ra file docs diễn ra mượt mà và chuẩn hóa.

## Steps

### Step 1: Guideline Audit
- Đọc file hướng dẫn trong `guidelines/project-level/`.
- Xác nhận với User: "Tôi sẽ bắt đầu thực hiện file [Tên file] theo chuẩn [X], bạn đồng ý chứ?"

### Step 2: Information Gathering
- Gửi bộ câu hỏi được tạo bởi skill `context-collector`.
- Tổng hợp câu trả lời và highlight những điểm cần làm rõ thêm.

### Step 3: Document Construction
- Viết file `.md` vào thư mục đích.
- Đảm bảo có mục lục (Table of Contents) và tuân thủ đúng Template 5 phần.

### Step 4: Consistency Check
- Kiểm tra xem file vừa viết có mâu thuẫn với các file Project-level đã tồn tại trước đó không.

### Step 5: Approval & Archive
- Trình User duyệt.
- Sau khi duyệt, đóng Task và lưu trữ.
